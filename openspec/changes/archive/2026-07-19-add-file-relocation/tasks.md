## 1. Domain and ports

- [x] 1.1 Add `Relocated bool` and `RelocateError string` fields to `domain.FileRecord` (`internal/domain/tracking.go`)
- [x] 1.2 Add a `Relocator` port to `internal/usecases/ports.go`: `Relocate(ctx context.Context, path string, meta RelocateInput) (newPath string, err error)` and `Undo(ctx context.Context, currentPath, originalPath string) error` (best-effort move-back when recording a successful relocation fails — see design.md), plus a `RelocateInput` struct carrying artist/album/title/trackNumber (everything needed to compute the destination — no cover art/lyrics needed here, unlike `TagInput`)
- [x] 1.3 Add `TrackingStore.RecordRelocation(ctx context.Context, oldPath, newPath string) error` to the `TrackingStore` interface (updates `path`, `relocated`, `relocate_error` in one statement) — plus `RecordRelocationFailure` as a separate method (failure doesn't touch `path`, success does, so combining them would need a nullable newPath)
- [x] 1.4 Add a `StatusChecker` interface to `internal/usecases/job_manager.go`: `type StatusChecker interface { Status() JobStatus }` — every existing manager (`RefreshManager`, `IdentifyManager`, `EnrichManager`, `TagManager`) already satisfies this via its existing `Status()` method, no changes needed on their side beyond satisfying the interface implicitly
- [x] 1.5 **Found via testing (task 8.2)**: `ErrRefreshInProgress`, `ErrRelocateInProgress`, `ErrTagInProgress`, etc. are all literally the same value (`= ErrJobInProgress`), so `errors.Is` can never distinguish "blocked by a different job" from "this job's own busy state" once one manager needs to report both. Added two new, genuinely distinct sentinels — `ErrBlockedByScan` (`relocate_manager.go`) and `ErrBlockedByRelocate` (`refresh_manager.go`) — instead of reusing the aliased ones for the cross-manager cases
- [x] 1.6 **Found via testing (task 8.2)**: writing tags changes a file's own size/mtime on disk, but nothing updated the tracking store's stored baseline — so the very next scan (including the automatic one at every server startup) saw the file as "changed", reset its status to `new`, and wiped its resolved metadata, while `tagged` stayed true (an inconsistent state). Added `TrackingStore.RecordFileStat(ctx, path, size, modTime) error` to `ports.go` to fix this

- [x] 2.5 Implement `Undo(ctx, currentPath, originalPath string) error` as a bare `os.Rename(currentPath, originalPath)` — no sanitization or `MkdirAll`, since the original directory already existed
- [x] 2.6 Verify path sanitization against real filenames containing each of `\ / : * ? " < > |` (a throwaway scratch check is fine, matching the pattern used to verify `go-taglib` behavior in the tagging change) — confirmed against real problematic names (`AC/DC`, `Guns N' Roses: Greatest Hits?`, `"Weird Al" Yankovic`, a synthetic string containing all 9 prohibited characters) via a throwaway Docker run
- [x] 2.7 **Found via testing (task 8.4)**: the destination-collision check also fired when a file's computed destination was itself (e.g. relocating an already-correctly-placed file again), since `os.Stat(dest)` trivially succeeds against the file's own current location. Fixed by checking `filepath.Clean(dest) == filepath.Clean(path)` first and returning success as a no-op in that case, before any collision check or filesystem call
- [x] 2.8 **Requested after initial implementation**: prefix the album directory with the release year, e.g. `2004 - Dead Letters`, omitted (not a placeholder) when the release has no usable year. Added `Year int` to `RelocateInput` (populated from `FileRecord.Year` in `RelocateFile.Relocate`) and an `albumDirName(year int, album string) string` helper in `relocator.go` used in place of the bare `sanitizeSegment(album)` call
- [x] 2.9 **Requested after initial implementation**: after a successful move, remove the file's now-possibly-empty original directory and any now-empty ancestor directories, up to (never including) `musicRoot`. Added `removeEmptyDirs(dir string)` to `relocator.go`, called after a successful `os.Rename` in both `Relocate` and `Undo`; treats a directory containing only recognized OS junk files (`.DS_Store`, `Thumbs.db`, `desktop.ini`, `._*` AppleDouble sidecars) as empty and deletes that junk along with the directory; stops (without erroring) at the first non-empty ancestor or at `musicRoot`, bounded by a `strings.HasPrefix` check against the cleaned root so it can never walk above it; failures are silently tolerated since this is best-effort cleanup, not part of relocation's correctness
